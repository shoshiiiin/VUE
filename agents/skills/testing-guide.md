# Testing Guide

## Beschreibung

Leitfaden zum Schreiben und Ausführen von Tests für dieses spezifische Projekt [10].

## Workflow

1. **Setup:**
   - Prüfe, ob die Datenbank (z. B. Supabase) oder notwendige Services laufen [11].
2. **Test-Erstellung:**
   - Wenn neuer Code geschrieben wurde: Schreibe IMMER einen begleitenden Test.
   - Nutze das Framework [Hier dein Framework einfügen, z.B. Jest/Vitest].

3. **Validierung:**
   - Führe `npm test` aus.
   - Der Code darf erst als "fertig" markiert werden, wenn alle Tests grün sind [5].
